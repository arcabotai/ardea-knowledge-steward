# FIPs and proposals

Governance material should be labelled by status:

- verified live rule
- draft proposal
- active discussion
- inferred ecosystem direction
- unknown / needs source

Do not present draft tokenomics, governance mechanisms, validator rules, or bridge/security changes as final unless an authoritative current source confirms them.

For large protocol changes, insist on design docs, threat models, green CI, CODEOWNERS, security policy, and split review units.
