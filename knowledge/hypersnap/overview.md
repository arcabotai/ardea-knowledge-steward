# Hypersnap

Hypersnap is best explained as an independent Farcaster-compatible/fork ecosystem, not merely a new skin for the Farcaster app.

It is concerned with:

- Snapchain/Farcaster-style protocol data and nodes.
- Builder and node-operator infrastructure.
- Public memory: forums, support answers, provenance-aware discussion, and governance context.
- Reducing dependency on farcaster.xyz and hosted compatibility APIs over time.

# Layer map

| Layer | Meaning |
|---|---|
| Farcaster protocol | The social protocol family and message/account model. |
| Snapchain | Protocol data and node infrastructure for Farcaster-style messages. |
| Hypersnap | Independent fork-compatible ecosystem and public infrastructure direction. |
| App/client | A user interface that may read/write protocol data. |
| Local metadata | Search indexes, moderation labels, tags, drafts, and cache. |

# Guidance

When explaining Hypersnap, separate compatibility from dependency. Compatibility with the Farcaster protocol can coexist with independence from farcaster.xyz-hosted surfaces.
