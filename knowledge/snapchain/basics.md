# Snapchain basics

Snapchain is the node/data infrastructure layer in this ecosystem. It is where node operators, sync state, message submission, shard lag, peers, and protocol data availability matter.

# Useful mental model

- A client is not the chain.
- A hosted API is not the protocol.
- Running a process is not the same as running a healthy synced node.
- Indexers and forums can add local metadata, but protocol messages should remain provenance-visible.
